# Основные принципы работы с логами

## Когда применять

- При добавлении новых логов в iOS-приложение
- При рефакторинге существующего кода с логированием
- При отладке сложных сценариев с помощью логов
- При логировании ошибок сетевых операций, сохранения данных, авторизации

## Кратко

- **Только** русский язык в логах
- **Добавляй контекст** - что именно произошло
- **Используй переменные** для технических деталей через `\(variable)`
- **Используй правильный уровень** (error, info, debug)

## 1. Русский язык для всех логов
- **Все логи пишутся на русском языке** - для удобства разработчиков

## 2. Примеры логов

См. полные примеры в [references/EXAMPLE.md](references/EXAMPLE.md).

Основные категории:
- **Ошибки** - критические ошибки (сеть, сохранение, авторизация)
- **Информация** - важные события (успешные операции, переходы)
- **Отладка** - детальная информация для разработки (время, количество элементов)

## 3. Уровни логирования

- **Error** - критические ошибки (сеть, сохранение, авторизация)
- **Info** - важные события (успешные операции, переходы)
- **Debug** - детальная информация для разработки (время, количество элементов)
